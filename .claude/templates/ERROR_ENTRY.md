# Шаблон записи ошибки в ERRORS.md

Используй этот формат при добавлении новой ошибки:

```markdown
**Дата:** YYYY-MM-DD
**Ошибка:** Краткое название ошибки
**Причина:** Почему это произошло (root cause analysis)
**Исправление:** Как была решена проблема (код, команды, подход)
**Не повторять:** Конкретный паттерн который надо избегать в будущем
```

## Примеры из проекта

### Хороший пример:
```
**Дата:** 2026-04-24
**Ошибка:** Edit tool не нашла текст для замены
**Причина:** Попытка включить номера строк в old_string
**Исправление:** Использовать только содержимое строк без номеров
**Не повторять:** Копировать точное содержимое из Read, но не номер строки
```

### Плохой пример:
```
**Дата:** 2026-04-24
**Ошибка:** Не сработало
**Причина:** Ошибка
**Исправление:** Переделал
```

## Когда добавлять
1. **Технические ошибки** — ReferenceError, TypeError, неправильный формат
2. **Ошибки процесса** — забыл прочитать файл перед Write, неправильный синтаксис
3. **Интеграционные** — конфликты с архитектурой, неправильный импорт

## Не добавлять
- Обычные исправления опечаток
- Успешно выполненные задачи
- Бизнес-логические изменения
