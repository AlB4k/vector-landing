# Чек-лист для новой фичи

Используй этот чек-лист перед `git push` для новой фичи:

## 📋 Code Quality
- [ ] Код соответствует CLAUDE.md (архитектура, стиль)
- [ ] Нет console.log/console.error в продакшене
- [ ] Импорты отсортированы и полные
- [ ] Нет `any` типов (если TypeScript)

## 🧪 Testing
- [ ] Функция протестирована локально
- [ ] Edge cases покрыты
- [ ] Нет регрессий в существующих фичах
- [ ] Работает в Chrome, Safari, Firefox (если фронтенд)

## 🔄 Build & Deploy
- [ ] `npm run build` проходит без ошибок и warning
- [ ] `npm run lint` (если есть) проходит
- [ ] Production build меньше dev

## 📚 Documentation
- [ ] CLAUDE.md обновлен если архитектурные решения
- [ ] Комментарии добавлены к сложной логике
- [ ] README.md обновлен если есть

## 🚀 Pre-Merge
- [ ] Ветка актуальна с main (`git pull origin main`)
- [ ] Commit message соответствует conventional commits
- [ ] Нет конфликтов при merge

## ✨ CMS-Specific (если касается CMS)
- [ ] Draft-state pattern соблюден
- [ ] Использована `interpolate()` для динамического текста
- [ ] Контент валидируется перед сохранением
- [ ] Защищенные роуты имеют auth проверку

## 🎨 UI-Specific (если касается интерфейса)
- [ ] Дизайн соответствует визуальной идентичности
- [ ] Адаптивен для мобильных (если требуется)
- [ ] Доступен для скринридеров (basic a11y)
- [ ] Нет магических числовых констант (использовать переменные)
