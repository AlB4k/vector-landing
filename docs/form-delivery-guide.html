Telegram Bot

Мгновенный прием заявок в мессенджер

1

Создайте бота в @BotFather

Найдите официального бота @BotFather. Отправьте /newbot, задайте имя и получите API Token (длинная строка с двоеточием).

2

Активируйте бота

Важно: Перейдите в созданного бота и нажмите «Запустить» (Start) или напишите любое сообщение. Бот не сможет отправить вам заявку, если вы не начали с ним диалог.

3

Получите ваш Chat ID

Скопируйте ваш Token и вставьте его в эту ссылку (без квадратных скобок):

https://api.telegram.org/botВАШ_ТОКЕН_ЗДЕСЬ/getUpdates

Откройте ссылку в браузере. Найдите объект "chat":{"id":XXXXXXXXX}. Это число и есть ваш Chat ID.

Настройка канала

Если вы хотите получать заявки в канал: добавьте бота в администраторы канала и укажите публичное имя канала (например, @my_channel) в поле ID в CMS.

EmailJS Service

Бесплатная отправка Email без бэкенда

1

Подключите почтовый сервис

В кабинете EmailJS перейдите в Email Services -> Add New Service (например, Gmail). Скопируйте Service ID.

2

Настройте шаблон (Template)

В разделе Email Templates создайте новый. В поле сообщения используйте переменные в двойных скобках:

{{from_name}} — Имя
{{phone}} — Телефон
{{organization}} — Компания
{{message}} — Текст
3

Получите API Keys

Перейдите в Account -> API Keys. Скопируйте Public Key. Теперь у вас есть все 3 ключа для CMS.

SMTP Server

Максимальный контроль через свой сервер

Этот метод требует наличия бэкенда (Node.js/PHP/Python), который примет POST-запрос от лендинга и отправит письмо через SMTP.

1

Разверните обработчик

Используйте файл docs/smtp-handler-example.js как основу. Настройте его на вашем сервере (например, через Vercel Functions или Express.js).

2

Свяжите фронтенд

В файле src/utils/formSender.js убедитесь, что путь в fetch('/api/send-mail') соответствует вашему реальному API.

Безопасность и хранение

Все секретные данные (Токены, Ключи, Пароли) хранятся исключительно в LocalStorage вашего браузера. Они не попадают в JSON-файлы контента и не загружаются в GitHub. Если вы смените компьютер или очистите данные сайта, настройки доставки нужно будет ввести заново.
